## Project Overview

Mie Ayam Lariska Web is a static website for a chicken noodle restaurant in Indonesia. It provides:
- Customer-facing menu with 24 items in 3 categories and real-time stock status
- 4 dynamic package recommendations from Firebase
- WhatsApp-based ordering system
- Admin dashboard for menu and packages management
- Image upload, drag-drop reordering, and auto-generate descriptions
- Firebase Realtime Database for data persistence

**Stack**: Vanilla JavaScript, HTML5, CSS3, Firebase, Playwright
**Pattern**: Static site with IIFE JavaScript modules
**Hosting**: Firebase Hosting

## Architecture Constraints

1. **Static files only**: No server-side functions or API endpoints
2. **Firebase client-side SDK**: All database operations from browser
3. **No build process**: Direct HTML/CSS/JS files
4. **No framework**: Vanilla JavaScript only
5. **Mobile-first**: Responsive design with CSS custom properties
6. **IIFE pattern**: All JavaScript modules use IIFE with `'use strict'`
7. **BEM naming**: All CSS classes follow BEM convention

## Security Notes

- Firebase credentials exposed in client-side code
- Admin credentials hardcoded: `lariska` / `lariska123`
- No server-side validation
- Public read/write Firebase rules for menu, packages, and stock data

```

│     price, tag, isFeatured, isActive,
│     whatsappMessage, order
│   }

stock/ (backward compatibility)
├── {item_id}: { status }
```
